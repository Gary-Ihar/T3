Тестовое задание
Требования
1. Необходимо создать HTML страницу с видеотегом (размер 640x360, для
управления воспроизведением использовать дефолтные элементы управления
видеотега), который будет воспроизводить контент, выбираемый пользователем из
выпадающего списка, расположенного на той же странице.

2. Ссылки на контент указаны в конце задания. Там три формата видео: MP4, HLS,
DASH. MP4 воспроизводится браузерами нативно, а для HLS и DASH используйте
сторонние библиотеки -- hls.js и dash.js соответственно.

3. Воспроизведение должно начинаться автоматически сразу после выбора единицы
контента без перезагрузки страницы. В случае если автоматическое
воспроизведение невозможно(здесь можно узнать, почему такое может быть
https://developer.chrome.com/blog/autoplay/), в консоли браузера не должно быть
сообщений об ошибках. Иными словами, вам нужно учесть политику автоплея
браузера.

4. Основная часть задания:
Процесс воспроизведения контента имеет разные состояния:
контента нет, видеотэг пуст (IDLE)
контент загружается, воспроизведение не начали (LOADING)
контент загрузился, готовы к воспроизведению (READY)
воспроизводим (PLAYING)
пауза (PAUSED)
перемотка (SEEKING)
буфферизация (BUFFERING)
воспроизведение окончено (ENDED)
Необходимо написать класс, который в качестве аргумента принимает ссылку на
видеотэг, и в любой момент времени может определить, в каком из перечисленных
выше состояний находится процесс воспроизведения.
Этот класс, обнаружив изменения состояния воспроизведения(например,
пользователь нажал паузу), должен сообщить об этом в консоль браузера. Для
состояния BUFFERING необходимо определить длительность буфферизации.
Этот класс должен одинаково точно работать со всеми тремя видео из задания,
корректно обрабатывать сценарий "досмотрели до конца и начали смотреть
сначала", корректно реагировать на выбор нового видео из выпадающего списка.

Не использовать ИИ

Результат должен состоять из двух частей: HTML-страница с разметкой, скриптами
загрузки hls.js, dash.js в /example/index.html и кода основного приложения в папке
/src

Основное приложение "Видеоплеер":
В конструкторе получает ссылку на видеотег.
Через метод load(url: string): void получает из HTML-страницы ссылку на
выбранное видео, и внутри себя определяет, как этот контент воспроизводить
- с помощью библиотек или силами браузера.
Автоматически начинает воспроизведение, если это возможно.
Определяет изменение состояния процесса воспроизведения я выводит его в
консоль. Для буфферизации определяет, сколько она длилась.
GUI для плеера не нужен, достаточно штатных контролов видеотега.
Использовать TypeScript.
Результат положить в открытый репозиторий на Github, приложить инструкции для
сборки и запуска.
Основной критерий оценки для нас это качество и чистота кода, чем ближе он
будет к приципам SOLID, тем лучше
